🔧新手焊接要点

线索

焊导线不焊排针，怎么焊好？

  • 先上锡，再贴合
  • 第三只手固定
  • 焊点套热缩管
  • 26AWG 软线好焊

本方案全部焊导线、不焊排针（保持低高度、贴平面板）。新手按这几条做不容易翻车：

1
两边先上锡
焊盘和线头各先熔一点锡（搪锡），再对上一碰即焊牢
2
第三只手
用焊接辅助夹固定板和线，腾出两只手拿烙铁+锡
3
先穿热缩管
焊前把热缩管先套到线上，焊完推过去吹热收紧
4
温度+时间
恒温 ~330℃，每点 ≤2 秒，焊点光亮饱满不拉尖
✓ 好焊点长这样：光亮、饱满、像小山包；轻拽线不动。灰暗、像球、一拽就掉 = 虚焊，重焊。
⚠ 最常见三个翻车：① 焊盘没上锡直接怼线 → 虚焊；② 烙铁停太久 → 焊盘脱落/烧坏元件；③ 忘了先穿热缩管 → 焊完才发现套不进去。
📌 一句话总结：两边先搪锡、第三只手固定、焊前先穿热缩管、每点 2 秒内。

5接线（核心）

分五部分：(A) 屏接主板、(B) 灯环接主板、(C) 电源链、(D) 电池电量（可选）、(E) 按钮（可选）。先别通电，全部焊好、调好 5V 再上电。

正极 / 电源 GND 地 数据/信号 数据/信号 ● 焊点（线落在引脚圆心）

(A) 屏幕 ↔ 主板（8 根线，平行直连）

⚠ 屏的 VCC 一定接 3.3V（3V3），千万别接 5V，会烧屏！

(B) 灯环 ↔ 主板（含 470Ω 电阻 + 470µF 电容）

⚠ 灯环 DIN 接的是主板 GPIO5。注意：很多 ESP32-S3 Nano 克隆板丝印写 D2 才是 GPIO5（丝印的「D5」往往不是 GPIO5）。本指南固件按 GPIO5 配置——接到丝印 D2 那个脚。灯不亮先怀疑这里。
💡 为什么灯环电源接锂电 3.7V，不接主板 5V？WS2812B 用 5V 供电时要求数据高电平门槛 3.5V，而主板 GPIO 只能输出 3.3V，正好卡边界。改用 3.7V 锂电直供后 3.3V 数据就在安全区，免电平转换芯片。代价仅是灯略暗，状态指示无感。
灯环脚接到中间元件线色
VCC（+）开关之后、升压之前的 3.7V（= MT3608 的 IN+）VCC↔GND 间并 470µF（长脚+朝VCC）红
GND（−）锂电负极 + 主板 GND（共地）—灰/黑
DIN主板 D2（=GPIO5）串 470Ω（靠灯环端）黄
⚠ 灯环 VCC 必须接「开关之后」的 3.7V（MT3608 IN+），不能接 TP4056 OUT+ / 电池正极！否则开关只切主板、切不掉灯环 → 关机后灯环有电没信号，显示乱码常亮。接到开关后的 IN+，开关一关主板和灯环一起灭。

(C) 电源链：电池 → 充电 → 开关 → 升压 → 主板

这是「有先后顺序」的部分，用分镜格一步一步看，再看整图：

1
电池 → TP4056
电池 +/− 焊到 TP4056 的 B+ / B−
2
TP4056 → 开关
OUT+ 接自锁开关一脚；OUT− 留作总负极
3
开关 → MT3608
开关另一脚 → MT3608 IN+；OUT− → IN−
4
先调 5V！
⚠ MT3608 单独通电，万用表拧到 5.0V
5
MT3608 → 主板
调好后 OUT+/OUT− → 主板 5V / GND

(D) 电池电量监测（可选，2× 2.2kΩ 分压）

💡 为什么要两个电阻？锂电满电 4.2V，但主板 ADC 最高只能量 ~3.3V，直接接会超量程甚至烧脚。两个等值电阻串联把电压砍一半（4.2V→2.1V），中间抽头给 ADC 读。固件已按 2:1 配置，不接也不影响其它功能（自动判定「无传感器」）。电阻无正负极，但抽头要接在两电阻中间。

(E) 实体按钮（可选，翻页 / 调暗）

💡 用什么零件：任意 2 脚自复位轻触开关（momentary tactile switch，按下导通、松开断开，无正负极）。淘宝搜「自复位按钮开关 2脚」或「6×6 轻触开关」。常见的 6×6mm 4 脚轻触开关也能用——它 4 个脚其实是两两内部相连的一组，取对角的两个脚即可。

接法：按钮一脚接 GPIO2（这块板丝印是 A1，不是 D1！低位 GPIO 标 A 开头），另一脚接 GND。固件用内部上拉，不按=高电平，按下接通到 GND=触发。短按翻页、长按调暗。不接也行——手机打开 cc-hud.local 网页有同样的「next page / dim」按钮。

⚠ 丝印对照（本 clone 板 = Arduino Nano ESP32 真排列）：GPIO1=丝印 A0、GPIO2=丝印 A1、GPIO5=丝印 D2。低位 GPIO 标 A0–A7，别照 GPIO 号去找「D+数字」。拿不准就用万用表通断档：从丝印脚量到 ESP32 芯片对应引脚确认。

6调 5V（MT3608）——上电前最关键一步

线索

为什么先调 5V？

  • 裸板出厂可能输出 12V+
  • 直接接主板 = 烧板
  • 万用表拧到 5.0V 再接
1
单独通电
MT3608 IN 接电池（经开关），OUT 先不接主板
2
表笔搭 OUT
万用表直流电压档，红表笔 OUT+、黑表笔 OUT−
3
拧电位器
慢拧蓝色电位器，读数调到 5.0V
4
断电再接
断电，OUT+/OUT− 接主板 5V/GND
⚠ 这步不能省、不能猜。没有万用表就别接升压——一旦输出过压，主板瞬间烧毁不可逆。
✓ 过关标准：万用表稳定读 4.9–5.1V 才能接主板。读数乱跳/偏高 → 继续拧电位器；拧到底还高 → 模块不对，换一个。
📌 一句话总结：MT3608 必须先单独通电、万用表拧到 5.0V，再接主板。

7刷固件（三条路）

线索

怎么把固件烧进去？

  • 首刷走 USB 串口
  • 之后 BLE / WiFi OTA 无线刷
  • 插电脑黑屏 = CDC 时序坑

三条路，按"什么情况用哪条"选：

方式什么时候用速度前提
① USB 串口第一次刷 / 彻底救砖~10sUSB 数据线 + 板 USB 数据正常
② BLE OTA无 WiFi / WiFi 刷不了时救砖~22s蓝牙权限 + 设备在广播
③ WiFi OTA日常迭代（最快）~2s 上传已配过 WiFi（见下）

决策树：第一次 → 走①把固件刷进去 → 用 push_wifi.py 配一次 WiFi → 之后都用③。③ 刷不动了 / 设备失联 → 用② BLE 救回来（BLE 不依赖 WiFi）。

# ① USB 首刷
pio run -e esp32s3_nano -t upload

# ③ WiFi OTA（日常，先 build 再传；mDNS 抽风就用 IP）
pio run -e esp32s3_nano_wifi -t upload
pio run -e esp32s3_nano_wifi -t upload --upload-port 192.168.x.x

# ② BLE 救砖（不需要 WiFi，需蓝牙权限）
python ota.py --address <设备UUID> \
  --firmware .pio/build/esp32s3_nano/firmware.bin --verbose
⚠ 坑1：USB-C 插电脑，屏背光亮但不显示。ARDUINO_USB_CDC_ON_BOOT=1 时插活动 USB 主机会在开机早期触发枚举，干扰 ST7789 初始化时序。本指南固件已把 displayInit() 提到 Serial.begin() 之前修掉。
⚠ 坑2：USB 首刷识别不到串口。这块板 USB 数据偶尔不稳。按住 BOOT(GPIO0) → 点一下 RST → 松开，强制进 ROM 下载模式（只有 RST 键时，用镊子把 GPIO0 短到 GND 替代 BOOT）。还不行就换条数据线（很多是充电线没数据）。
✓ 刷成功的标志：USB 看到 Writing... 100% + 自动重启；OTA 看到 100% / OK / Success，设备屏一闪回到 HUD。WiFi OTA 进行时屏幕会显示进度条。
⚠ 救砖口诀：WiFi 刷不进 / 设备 ping 不通 → 别慌，BLE OTA 一定能救（蓝牙独立于 WiFi）。跑上面 ② 那条即可。
📌 一句话总结：首刷 USB → 配 WiFi → 日常 WiFi 秒刷；刷挂了用 BLE 救（蓝牙不依赖 WiFi）。

8电脑端配置

线索

电脑怎么把数据推给盒子？

  • statusline 推额度
  • hooks 推状态
  • 首次推一次 WiFi 凭证

三步：配网 → 接状态栏 → 接 hooks。(脚本都在仓库 host/ 下，先 cd host 建好 .venv)

① 一次性配 WiFi（经蓝牙推 SSID/密码，存盒子 NVS，之后开机自动联网）：

.venv/bin/python push_wifi.py --address <设备UUID> \
    --ssid '你的WiFi名' --password '你的密码' --verbose

② 状态栏推额度（~/.claude/settings.json）—— 每次刷新把 5h/7d 额度、ctx、成本、增删行推给盒子，顺带每 10 分钟校时+推天气：

"statusLine": { "type": "command",
  "command": "/你的路径/cc_hud_esp32/host/cchud-statusline.sh" }

③ hooks 推状态（同文件 "hooks" 段）—— 把 AI 的 thinking/tool/idle/waiting 推给盒子，多会话自动聚合（waiting>tool>thinking>idle）：

"UserPromptSubmit": [{"hooks":[{"type":"command","command":".../cchud-hook.sh thinking"}]}],
"PreToolUse":  [{"matcher":"*","hooks":[{"type":"command","command":".../cchud-hook.sh tool"}]}],
"PostToolUse": [{"matcher":"*","hooks":[{"type":"command","command":".../cchud-hook.sh thinking"}]}],
"Stop":         [{"hooks":[{"type":"command","command":".../cchud-hook.sh idle"}]}],
"Notification": [{"hooks":[{"type":"command","command":".../cchud-hook.sh waiting"}]}]

仓库 host/settings.hooks.example.json 是可直接抄的完整模板，把路径换成你的。

💡 网页面板：同 WiFi 下手机/电脑开 http://cc-hud.local/，看实时额度/成本/状态/电量，拖亮度滑块实时调灯环（存 NVS、免重刷），「next page / dim」遥控按钮 = 实体按钮的零硬件替代，底部还有 固件版本 / 运行时长 / 内存。
✓ 配好的标志：跟 Claude Code 对话时，盒子屏数字会变、灯环随状态变色；ping cc-hud.local 通。额度没动？多半是 Claude 这会儿还没下发 rate_limits（订阅用户真用起来才有），正常。
📌 一句话总结：push_wifi 配网 → statusLine 推额度 → hooks 推状态；之后手机网页实时看+遥控。

Codex 接法——编辑 ~/.codex/config.toml：

notify = ["/你的路径/cc_hud_esp32/host/cchud-codex-notify.sh"]

Codex 每个 turn 结束会调它 → 灯环绿色脉冲闪三下（"该你看了"）。这个脉冲无视当前状态强制显示 ~1.8 秒，所以即便 Claude 在忙也看得到。

注意：Codex 的 notify 只支持一个程序。若你已有别的 notify（如 computer-use），用 cchud-codex-dispatch.sh 链式调用同时保留两者，再把 notify 指向它。

💡 想要 Codex 也显示"working"黄灯？Codex 没有"开始工作"事件，notify 只在答完时触发。要细粒度需开 Codex 的实验性 hooks（~/.codex/hooks.json + codex_hooks flag），把 pre/post 事件指向 cchud-hook.sh tool / idle。实验特性、Windows 不支持。
✓ Codex 接好的标志：在终端跑 codex exec "说你好"，Codex 答完那刻 → 灯环绿色闪三下。已实测通过。
📌 一句话总结：设备工具无关；Claude 走 statusLine+hooks 最完整，Codex 走 config.toml 的 notify 显示 turn 级状态，多工具会话自动聚合。

通电前 / 通电后逐项核对

  • 先断电查焊接
  • 再上电查显示
  • 最后查无线

① 通电前（万用表通断档，断电状态）

检查期望
屏 VCC ↔ 主板 3V3通；且 不与 5V 相通
灯环 VCC ↔ MT3608 IN+通（开关后的 3.7V，不是 TP4056 OUT+）
灯环 GND ↔ 主板 GND通（共地）
电容长脚 ↔ 灯环 VCC通（长脚=正极朝 VCC）
任意 VCC ↔ 任意 GND不通（通=短路，排查）
MT3608 OUT+ 电压单独通电拧到 5.0V 再接主板

② 通电后（先电池供电）

检查期望
屏点亮显示 HUD（标题/额度条/footer）
开关断开屏 + 灯环一起灭（不灭=灯环接错到开关前）
灯环空闲=绿色弧；推 state 测试=黄/红变化

③ 无线

检查期望
BLE 推额度屏数字更新、右上点变绿
ping cc-hud.local通（WiFi 已连）
手机开 cc-hud.local看到实时面板 + 翻页/调暗按钮
📌 一句话总结：断电先查「该通的通、该断的断、5V 调好」，上电查「屏亮 + 开关能关灯环」，最后查「BLE/WiFi 通」。

不亮 / 不显示 / 不更新？

  • 电池正常但 USB 不显示
  • 灯不亮的 4 个原因

症状原因解决
USB-C 供电屏亮但不显示，电池供电正常USB CDC 枚举干扰屏初始化时序刷本指南固件（displayInit 已前移）；或先用纯充电头（不走数据）
USB-C 插电脑识别不到串口线是充电线 / 没进下载模式换数据线；按住 GPIO0 再点 RST 进下载模式
灯完全不亮① DIN 接到的不是 GPIO5（丝印 D5≠GPIO5，应接 D2）
② 电容接反击穿，把 VCC 拉到 ~0.6V
③ 没共地
④ 3.7V 太低		① 改接丝印 D2
② 拆掉/换电容，长脚+朝 VCC
③ 灯环 GND 与主板 GND 直连
④ 给电池充电
只亮约 1/3 的灯固件灯数配成 8，实际 24用本指南固件（已配 24）
彗星转太快步进太短本指南固件已调到 ~2s/圈
屏显示但额度不更新Claude Code 暂未下发 rate_limits（新会话/长时间没调 API）正常现象；真正交互产生用量后 30s 内自动更新
电池电量不显示分压电阻没接（显示「无传感器」）接 2× 2.2kΩ 分压到 GPIO1（D 节）
电容鼓包/发烫电解电容接反立即断电，换新电容，长脚+朝 VCC
📌 一句话总结：USB 黑屏=CDC 时序、灯不亮先查 D2/电容极性/共地、额度不更新多是 Claude 还没下发数据。
